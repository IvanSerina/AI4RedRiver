# Synthetic Datasets

Synthetic datasets generated by introducing controlled variations into the historical data (2001–2012 period). They allow simulation of hypothetical or extreme scenarios beyond those observed historically, for stress-testing the water-release optimisation models.

## Generation Method

1. A time interval from the historical record was selected and mean values of the variables of interest were computed.
2. Random noise sampled from a Gaussian distribution (based on the standard deviation of the original data) was added to produce a realistic baseline.
3. Targeted modifications were applied — increasing or decreasing specific features by fixed percentages — to create scenario variants.

## File Naming Convention

```
synthetic_data_2001_2012_<modification>.csv
                │         └── Description of the applied change
                └── Source period
```

### Modification codes

| Code pattern | Meaning | Example |
|---|---|---|
| `minus_XX_inflow` | Inflow reduced by XX% | `minus_01_inflow` → −10% inflow |
| `plus_XX_inflow` | Inflow increased by XX% | `plus_02_inflow` → +20% inflow |
| `minus_XX_demand` | Demand reduced by XX% | `minus_01_demand` → −10% demand |
| `plus_XX_demand` | Demand increased by XX% | `plus_02_demand` → +20% demand |
| `minus_XX_storage` | Storage reduced by XX% | `minus_01_storage` → −10% storage |
| `plus_XX_storage` | Storage increased by XX% | `plus_015_storage` → +15% storage |
| `minus_XX_in_minus_YY_storage` | Inflow −XX% **and** storage −YY% | Combined scenario |
| `minus_XX_in_plus_YY_demand` | Inflow −XX% **and** demand +YY% | Combined scenario |
| `plus_XX_in_plus_YY_demand` | Inflow +XX% **and** demand +YY% | Combined scenario |
| `plus_XX_in_plus_YY_storage` | Inflow +XX% **and** storage +YY% | Combined scenario |
| `plus_XX_demand_minus_YY_storage` | Demand +XX% **and** storage −YY% | Combined scenario |

> **Note:** `01` = 10%, `005` = 5%, `015` = 15%, `02` = 20%, `020` = 20%.

### Special files

| File | Description |
|------|-------------|
| `synthetic_data_2001-2012-02.csv` | Gaussian-noise baseline (includes `Day_month` column) |
| `synthetic_data_2015-2020-03.csv` | Baseline generated from the 2015–2020 period |

## Columns

All files share the same schema as `Historical_data.csv`:

| Column | Unit |
|--------|------|
| `Date` | — |
| `Input_Hoa_Binh` | m³/s |
| `Yen_Bai` | m³/s |
| `Vu_Quang` | m³/s |
| `Evaporation` | mm/day |
| `Demand` | m³/s |
| `H_Up (m)` | m |
| `Energy_demand` | MWh |
| `Qtu_(m3/s)` | m³/s |
| `Qbot_(m3/s)` | m³/s |
| `Qspill_(m3/s)` | m³/s |
| `Q_Sơn Tây` | m³/s |
| `Q_Hà Nội` | m³/s |
| `H Hà Nội` | m |

> Dates in synthetic files are shifted to a fictitious year (e.g. 2077) to distinguish them from real observations.

## Usage

These datasets are used as input to the dashboard planner (see [`dashboard_redriver/`](../../dashboard_redriver/README.md)) to evaluate how the optimisation performs under altered conditions.
